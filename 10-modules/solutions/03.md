# Solution to Chapter 10 - Problem 3: Circular Dependencies

I'm not the most entitled person to talk about this. That said, the difference resides on the How and When `require` handle the import. 

The "How" involves the utilization of an object that will be available when the function gets used. That way, being calling in a circular fashion won't present many problems, since the object already contains the required logic that it may need. 

That said, the "When" part represent the usage time, which may vary inside the script, when ES Module is used the first step is to import all the required pieces of code, when facing a circular reference this strategy eventually enters in a deadlock state, calling functions non-stop until the runtime stack overflows 🤯.

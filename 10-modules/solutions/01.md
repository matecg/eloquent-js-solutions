# Problem 1 - A Modular Robot

My solution on chapter 07 involved 4 scripts:

- `helpers.js`
- `index.js`
- `robots.js`
- `village-state.js`

The main file is `index.js` which has no definition of its own. It is simply used to load different modules and to consume them in the way that builds the program. So `index.js` imports from all 3 other modules.

Then, I've implemented the `village-state.js` that is responsible for storing the `VillageState` class only. This module also requires code from `helpers.js` so an import statement was used to.

The `robots.js` file contains the logic for all robot functions, being the only exception the robot that was designed in Problem 02. Once again, this module also require pieces of code from `helpers.js`.

Finally, `helpers.js` is the least meaningful module in terms of consistency, as it houses several different functions and constants that were not big enough to be segregated into their on file. All modules depend on it, which usually isn't something that is desirable.

One modification that this basic architecture could receive would be to remove constants, like `VILLAGE_ROUTE` and `ROADS` and store them on their own file, called `constants.js`.
